# AI Job Radar 工程方案

## 1. 产品定位
AI Job Radar 是接单平台检索与机会评分工具，不是绕过平台规则的自动接单机器人。核心价值是把单子收集、归一化、打分、解释、投标草稿和交付流程自动化。

## 2. 标准工程阶段

### 阶段A：MVP展示版
- 本地样例数据
- 静态前端评分与推荐
- 平台连接状态矩阵
- 投标草稿和交付清单
- CSV导出

### 阶段B：本地后端
- FastAPI服务
- SQLite存储
- 导入JSON/CSV
- 评分器API
- OpenAPI schema给GPT Actions使用

### 阶段C：平台连接器
优先级：
1. 官方API：Upwork、Freelancer
2. 用户导出CSV：Fiverr、PeoplePerHour、猪八戒、一品威客
3. 浏览器扩展：只读取当前用户可见页面，不模拟登录、不绕验证码
4. 手动粘贴：作为所有平台兜底

### 阶段D：GPT/Codex协作
- GPT：分析、评分、解释、生成投标草稿
- Codex：根据交付指令生成代码、文档、PPT大纲、脚本文件
- 人工确认：投标、发送消息、接受合同、收付款

### 阶段E：生产化
- 凭据加密
- 审计日志
- 速率限制
- 任务调度
- 错误告警
- 插件/MCP打包

## 3. 评分规则
- AI自主度：能不能由GPT/Codex生成主体内容
- 预计时长：是否能在2小时内完成
- 人工参与：是否需要大量沟通、设计审美、生产权限、客户后台
- 风险：版权、隐私、平台规则、虚假承诺、违法违规
- 预算：金额是否覆盖时间成本
- 客户信誉：评价、付款验证、需求清晰度
- 交付物清晰度：是否明确文件、格式、修订次数

## 4. 不自动化的动作
- 自动投标
- 自动花费平台Connects/积分
- 自动发送客户私信
- 自动接受合同
- 自动收付款
- 自动使用客户隐私数据
- 自动提交众测/标注结果

这些动作必须二次确认，因为它们影响账号安全、平台信誉、合同义务或个人隐私。
